# MCP 2026-07-28 — the stateless protocol (the `modern/` package)

This server is **dual-era**: it speaks both the legacy `initialize`-handshake
protocol (2025-11-25 and earlier, via FastMCP) and the new **2026-07-28**
stateless revision (implemented from scratch in [`modern/`](../../modern/) for
teaching). One HTTP endpoint serves both; each POST is classified by its
protocol markers and routed to the matching pipeline.

> **Spec:** <https://modelcontextprotocol.io/specification/draft> — the next
> MCP revision. On the wire the version string is **`2026-07-28`** (never
> `"draft"`). RC locked 2026-05-21; final publishes 2026-07-28.
>
> **Changelog:** <https://modelcontextprotocol.io/specification/draft/changelog>

## What changed, and where this server implements it

| Area | Change (SEP) | Spec page | Implemented in |
|---|---|---|---|
| **Stateless core** | No `initialize`/`initialized`; per-request `_meta` carries protocolVersion, clientInfo, clientCapabilities (SEP-2575) | [basic/index](https://modelcontextprotocol.io/specification/draft/basic/index), [basic/versioning](https://modelcontextprotocol.io/specification/draft/basic/versioning) | [`modern/meta.py`](../../modern/meta.py) |
| **`server/discover`** | Replaces `initialize` for capability discovery; servers MUST implement (SEP-2575) | [server/discover](https://modelcontextprotocol.io/specification/draft/server/discover) | [`modern/dispatcher.py`](../../modern/dispatcher.py) |
| **MRTR** | Server-initiated requests (sampling/elicitation/roots) replaced by `resultType: "input_required"` + `inputRequests`/`inputResponses`/`requestState` (SEP-2322) | [basic/patterns/mrtr](https://modelcontextprotocol.io/specification/draft/basic/patterns/mrtr) | [`modern/mrtr.py`](../../modern/mrtr.py), [`modern/context.py`](../../modern/context.py) |
| **`subscriptions/listen`** | Replaces `resources/subscribe`/`unsubscribe` and the HTTP GET stream; opt-in filter, ack, `subscriptionId` tagging (SEP-2575) | [basic/patterns/subscriptions](https://modelcontextprotocol.io/specification/draft/basic/patterns/subscriptions) | [`modern/broker.py`](../../modern/broker.py) |
| **Streamable HTTP** | One POST endpoint; required `Mcp-Method`/`Mcp-Name` headers + `x-mcp-header` (SEP-2243); no sessions (SEP-2567); no SSE resumability (SEP-2575) | [basic/transports/streamable-http](https://modelcontextprotocol.io/specification/draft/basic/transports/streamable-http) | [`modern/http.py`](../../modern/http.py) |
| **CacheableResult** | Required `ttlMs`/`cacheScope` on discover + list + read results (SEP-2549) | [server/utilities/pagination](https://modelcontextprotocol.io/specification/draft/server/utilities/pagination) | [`modern/registry.py`](../../modern/registry.py) |
| **Error-code policy** | `-32020` HeaderMismatch, `-32021` MissingRequiredClientCapability, `-32022` UnsupportedProtocolVersion; resource-not-found `-32002`→`-32602` | [basic/index#error-codes](https://modelcontextprotocol.io/specification/draft/basic/index) | [`modern/errors.py`](../../modern/errors.py) |
| **Authorization** | RFC 9728 PRM + bearer validation; RFC 9207 `iss`; CIMD; `application_type` in DCR (SEP-2468/837/2352) | [basic/authorization](https://modelcontextprotocol.io/specification/draft/basic/authorization) | [`modern/auth/`](../../modern/auth/) |
| **Skills extension** | `skill://` resources + `skill://index.json` (with digests) + `resources/directory/read` (SEP-2640) | [SEP-2640](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/2640), [agentskills.io](https://agentskills.io/specification) | [`modern/skills.py`](../../modern/skills.py), [`skills/`](../../skills/) |
| **Tasks extension** | `io.modelcontextprotocol/tasks`: polling `tasks/get`/`tasks/update`/`tasks/cancel`; no `tasks/list`/`tasks/result` (SEP-2663) | [docs/extensions](https://modelcontextprotocol.io/docs/extensions/overview) | [`modern/tasks_ext.py`](../../modern/tasks_ext.py) |
| **Deprecations** | Sampling, Roots, Logging deprecated (SEP-2577) but still functional through the migration window | [specification/draft/deprecated](https://modelcontextprotocol.io/specification/draft/deprecated) | delivered via MRTR / per-request `logLevel` |

## Running it

```bash
# Legacy protocol over stdio (FastMCP) — unchanged
VIRTUAL_LIBRARY_TRANSPORT=stdio just dev

# 2026-07-28 stateless protocol over stdio
VIRTUAL_LIBRARY_TRANSPORT=stdio-modern uv run python server.py

# Dual-era Streamable HTTP (one endpoint serves both eras)
VIRTUAL_LIBRARY_TRANSPORT=http VIRTUAL_LIBRARY_ALLOW_INSECURE_HTTP=true uv run python server.py

# ...with the built-in educational authorization server (full OAuth 2.1 locally)
VIRTUAL_LIBRARY_TRANSPORT=http VIRTUAL_LIBRARY_ALLOW_INSECURE_HTTP=true \
VIRTUAL_LIBRARY_MODERN_AUTH_ENABLED=true VIRTUAL_LIBRARY_DEMO_AS_ENABLED=true \
uv run python server.py
```

### Try it with `curl`

```bash
# server/discover — the replacement for initialize
curl -s -X POST http://127.0.0.1:8080/mcp \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  -H 'MCP-Protocol-Version: 2026-07-28' \
  -H 'Mcp-Method: server/discover' \
  -d '{"jsonrpc":"2.0","id":"d","method":"server/discover","params":{"_meta":{
       "io.modelcontextprotocol/protocolVersion":"2026-07-28",
       "io.modelcontextprotocol/clientInfo":{"name":"curl","version":"1.0"},
       "io.modelcontextprotocol/clientCapabilities":{}}}}'
```

The companion client (`../../../mcp-client-learning`) speaks this protocol and
auto-detects the era; `uv run mcp-client discover --server http://127.0.0.1:8080/mcp`
prints the full capability set, extensions included.

## Educational caveats

- The `modern/` package is written to be **read**, not to be a production SDK.
  It deliberately reimplements the wire protocol by hand (the official
  `mcp==2.0.0b1` SDK is where you'd get 2026-07-28 support for real).
- [`modern/auth/demo_as.py`](../../modern/auth/demo_as.py) is a *toy*
  authorization server so the whole OAuth flow runs on one laptop. It is
  in-memory and unhardened — never expose it.
- Deployment is intentionally out of scope.
